# 도면 기반 부재 카운팅 룰북

작성일: 2026-05-22
적용 범위: 철골 부재 (기둥·보) 1단계 카운팅
대상 도면 형식: DXF (modelspace)

---

## 1. 도메인 지식

### 1.1 적산 단계 개요
- 개수 → 규격 → 길이 → 단위중량 곱 → 총중량(KG)
- 1단계 목표: 개수 카운팅, 오차율 5% 이내

### 1.2 부재 부호 명명 체계
- 회사·도면 작성 도구마다 명명 다름
- 도면별 명명 예시 (도면1·2·4의 MC/SC/SG 계열, 도면3의 C/P 슬래시 결합, 도면5의 C 단순형 등)

### 1.3 일람표란 무엇인가
- "이 도면에 쓰는 부호 종류와 규격을 정리한 표"
- 도면 한 귀퉁이에 위치, 부호 종류별로 1~2번씩 등장
- 부재 실배치(본체)와 같이 카운트하면 안 됨

### 1.4 규격 안내 텍스트
- "부호 + 공백/하이픈/줄바꿈 + 규격" 형태 (예: "SC1 H-350x175x7/11")
- 부재 본체가 아니라 사양 설명 → 카운트 제외

### 1.5 적산 제외 부재
- BR1·BR2 (가새), PURLIN, DS (데크 슬래브), PS, WG1, CRG1 — 적산 대상 아님

### 1.6 톨로런스 기준 및 한계
- 현재: 정답 ≤ 5는 ±1, > 5는 상대오차 5% 이하
- 한계: 경계(5→6)에서 갑자기 빡빡해지는 비대칭, 일람표를 검출하고도 ±1 통과시키면 검출 의미 약화
- 2단계 진입 시 정확 일치 기준 재검토 예정

---

## 2. 부호 인식 룰

### 2.1 화이트리스트 기반 매칭
- 도면별 부호 목록(화이트리스트)으로 후보 텍스트 필터링
- 정답지 기반으로 도면별 화이트리스트 정의

### 2.2 부분 일치 (부호+규격)
- "BR2 L-80X80X7" 같이 규격이 붙은 텍스트도 "BR2"로 매칭
- 단, "MC10"이 "MC1"로 오매칭되지 않도록 부호 뒤 첫 글자가 숫자면 매칭 거부

### 2.3 슬래시 결합 표기
- "C1/P1" 같은 결합 표기는 본체 부호로 인정
- 의미: "C는 컬럼, P는 베이스플레이트(또는 파일링)" — 한 부재가 둘을 동시에 표현
- 회사마다 차이. 도면3·5에서 발견됨

### 2.4 도면별 화이트리스트 외 부호
- 기초(F·FS·MF), 철근(HD·D·SR·SBR), 상세참조(DS·DT), 두께표기(THK), 통심선(X·Y) — prefix 기반 자동 제외
- H-숫자 패턴 (예: H-200x100) — 규격 표기 자동 제외

---

## 3. 카운트 제외 규칙 — 3가지 신호

### 3.1 신호 1 — 텍스트 크기(height) 필터

**원리**: 도면의 텍스트는 *부재 실표기*와 *주석·오탐*이 다른 글자 크기로 그려진다. 둘 사이의 갭이 명확하면 임계값 한 줄로 거른다.

**도면 유형별 패턴** (이게 룰북의 핵심):
- **유형 A** (도면1·2): 본체 = 큰 글자, 주석 = 작은 글자 → "X 이상만 카운트"
- **유형 B** (도면3): 본체 = 가장 작은 글자, 일람표·보 부호 = 더 큰 글자 → 필터 적용 불가 (필터 미적용)
- **유형 C** (도면5): 본체 ↔ 일람표 갭이 좁음 (34.6) → 분리 불안정, 필터 적용 시 위험

**자동화 한계**:
- 어느 height 무리가 진짜 부재인지는 height 분포만으로 결정 불가
- 외부 신호 (정답지·LLM·비전) 없이는 도면별 수동 설정 필요
- 새 도면 추가 시 height 임계값 한 줄은 사람이 결정해야 함

### 3.2 신호 2 — 일람표 영역 자동 검출

**원리**: 일람표는 "표" 정의상 좁은 영역에 부호 종류가 모이고, 각 부호가 1~2번씩만 등장한다.

**검출 알고리즘** (의사코드):
1. 자유 텍스트(TEXT/MTEXT) 좌표를 격자로 분할
2. 점이 있는 격자칸을 8방향 인접으로 연결 영역 구성
3. 영역별 판정: 부호 종류 ≥ 4 종 AND 부호당 최대 등장 ≤ 2 회 → 일람표

**왜 자유 텍스트만 입력으로 쓰나**: 일람표는 자유 주석 텍스트(TEXT/MTEXT)로, 부재 본체는 블록 인스턴스(INSERT/ATTRIB)로 그려지는 게 일반적. 이건 DXF 구조상의 차이라 회사 무관.

**예외 — 본체도 자유 텍스트인 경우**:
- 도면3 유형에서는 본체 부재가 자유 텍스트로 그려짐 → 일람표 검출 입력을 오염
- 보정 룰: 부호당 자유 텍스트가 *일정 수* 초과면 본체로 간주해 검출 입력에서 제외 (일람표는 부호당 1~2개씩만 등장한다는 정의 활용)

**자동 판정 룰**: 일람표 영역 1곳 이상 검출 → 신호 2 ON.

**도면 유형별 적용**:
- 유형 A (도면1·2): 일람표가 *물리적으로 있어도* height 필터로 입력 단계에서 잘림 → 검출 0곳 → 신호 2 자동 OFF (본체 카운트엔 영향 0, 필터가 일람표 부호도 같이 거름)
- 유형 B (도면3): 본체가 자유 텍스트라 보정 룰 필요. 자동 판정이 실패할 수 있어 *수동 강제 ON* 필요
- 유형 C·D (도면4·5): 일람표가 깨끗하게 자유 텍스트로 분리 → 자동 ON 정상 작동

### 3.3 신호 3 — 규격 안내 텍스트 제외

**원리**: "부호 + 공백/하이픈/줄바꿈 + 규격" 형태 (예: "SC1 H-350x175x7/11", "SC1\\PH 350x175x7/11") 는 카운트 제외.

**자동 판정 룰**: 규격형 텍스트 수 ≥ 화이트리스트 부호 수 × 0.3 → 신호 3 ON.

**보호 룰**: 어떤 부호가 "단독형 0개 + 규격형 1개 이상"이면 신호 3 ON 시 그 부호 카운트가 0이 됨 → 신호 3 자동 OFF 강제. (도면1 BR2가 이 경우)

## 4. 도면 유형별 룰 적용 가이드

각 유형별로 (1) 본체 표기 방식 (2) 일람표 위치 (3) 적용 신호 조합 (4) 주의사항을 다음 형식으로 작성:

### 4.1 유형 A — 큰 글자=본체, 작은 글자=주석 (도면1·2)
- 본체 표기: INSERT 블록 또는 큰 글자 자유 텍스트
- 일람표: 작은 글자 자유 텍스트 (height 필터로 자동 제외)
- 신호 조합: 신호 1 ON, 신호 2 OFF (자동), 신호 3 OFF (자동)
- 주의: 도면별 height 임계값 절대값 다름 (도면1: 177, 도면2: 302). 전역 고정 불가.

### 4.2 유형 B — 블록 본체 + 자유 텍스트 일람표 (도면4)
- 본체 표기: INSERT 블록의 ATTRIB
- 일람표·규격 안내: 자유 텍스트
- 신호 조합: 신호 1 OFF, 신호 2 ON (자동), 신호 3 ON (자동)
- 주의: DXF 자체에 부재가 좌표 중복으로 그려진 경우 있음 → 적산 전문가 검수 필요

### 4.3 유형 C — 전부 자유 텍스트 + 슬래시 결합 본체 (도면3)
- 본체 표기: 자유 텍스트 슬래시 결합 (C1/P1·C2/P2 등)
- 일람표: 같은 자유 텍스트지만 부호 단독형 (C1·C2)
- 신호 조합: 신호 1 OFF (height 갭 역방향), 신호 2 *수동 강제 ON*, 신호 3 자동 ON (슬래시 결합 부산물)
- 주의: 자동 판정이 신호 2를 잘못 판단할 수 있음 (보정 룰 누락) → 수동 강제 필요

### 4.4 유형 D — 슬래시 결합 본체 + 깨끗한 일람표 (도면5)
- 본체 표기: 자유 텍스트 슬래시 결합
- 일람표: 자유 텍스트 부호 단독형, 본체와 명확히 분리
- 신호 조합: 신호 1 OFF (갭 좁음), 신호 2 ON (자동), 신호 3 ON (자동)
- 주의: 유형 C와 비슷해 보이지만 자동 판정이 정확 → 수동 개입 불필요

## 5. 의사결정 원칙

### 5.1 AI는 결정만, 도구는 측정만
- LLM/AI는 "어떤 룰을 적용할지" 결정
- 룰 자체의 카운팅·검출은 결정론적 코드로 수행
- 같은 입력 → 같은 출력 보장

### 5.2 회귀 안전망 절대 조건
- 새 룰 추가가 기존 도면 카운트를 깨뜨리면 안 됨
- 모든 변경을 회귀 테스트 뒤에 둠

### 5.3 사전 진단 → 룰 변경 순서
- 진단 데이터 없이 룰 변경 금지
- 새 도면 추가 시 (1) 텍스트 분포 진단 → (2) 신호 적용 시뮬레이션 → (3) 룰 변경 → (4) 회귀 검증

### 5.4 솔직한 한계 인정
- 안 풀리는 건 안 풀린다고 보고
- 임의 임계값 박지 않음. 외부 신호 없이 안 풀리는 결정이 있음을 인정

### 5.5 보편 룰 우선
- 도면별 특수 룰 최소화
- 회사·도면 무관 신호 (DXF 구조, 텍스트 패턴) 우선
- 도면명 하드코딩 금지

## 6. 알려진 한계

### 6.1 분리 TEXT 인코딩 미해결
- 일부 도면에서 부호 'SC1' 이 'SC'와 '1' 두 글자로 분리되어 익명 블록 안에 저장됨
- 룰 기반 결합은 다른 도면 회귀 위험 → 1단계 보류
- 2단계 또는 VLM 도입 시 재검토 예정

### 6.2 좌표 중복
- DXF 자체에 부재 엔티티가 같은 좌표에 2개씩 겹쳐 그려진 경우 존재
- 자동 제거는 진짜 인접 부재와 구분 불가 → 적산 전문가 검수 위임

### 6.3 신호 1 자동화 구조적 불가
- height 무리 구조가 도면마다 다름 (정방향·역방향·갭 좁음)
- "가장 큰 갭" 같은 일반 알고리즘은 특정 도면 유형에서 본체를 죽임
- 외부 신호 (정답지 합산·LLM·비전) 없이는 도면별 수동 설정 필요

### 6.4 자동 판정의 구조적 불일치 (도면3 특수 케이스)
- 일람표 검출 입력 전처리가 *자동 판정 경로*와 *실제 차감 경로* 사이에 다를 수 있음
- 도면3 유형(본체가 자유 텍스트)에서만 증상 발현
- 다른 도면에선 비재현 → 수동 강제 ON으로 우회

### 6.5 톨로런스 5% 룰의 한계
- 정답 ≤ 5는 ±1, > 5는 5% 기준
- 정답 6 부근에서 갑자기 빡빡해지는 비대칭
- 일람표 1곳을 검출하고도 ±1로 통과시키면 검출 의미 약화
- 2단계 진입 시 정확 일치 기준으로 강화 필요

## 7. 새 도면 추가 시 진단 절차

새 도면을 추가할 때 다음 순서로 진단:

1. **텍스트 height 분포 확인**
   - 부호 후보 텍스트의 height 값 분포
   - 본체 ↔ 주석 갭이 명확한지 / 본체가 큰 글자인지 작은 글자인지

2. **부호 표기 방식 확인**
   - modelspace TEXT/MTEXT vs INSERT 블록 vs ATTRIB
   - 슬래시·하이픈·공백 결합 표기 유무

3. **일람표 영역 검출 시뮬레이션**
   - 자동 검출 결과 (영역 수, 포함 부호)
   - 본체가 자유 텍스트인 경우 입력 오염 여부 확인

4. **신호 3 자동 판정 시뮬레이션**
   - 규격형 텍스트 빈도
   - 보호 룰 발동 부호 확인

5. **카운트 시뮬레이션**
   - 각 신호 조합별 결과
   - 정답지와 비교

6. **유형 판정**
   - 유형 A/B/C/D 중 어느 것에 가까운지
   - 새 유형이면 보편 룰로 처리 가능한지 검토

7. **수동 개입 결정**
   - 자동 판정이 정답에 도달하면 자동만 사용
   - 자동이 잘못 판단하면 수동 강제 ON/OFF로 보정 (비상 수단)
