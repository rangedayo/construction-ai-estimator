## B. 정책 P 정의 및 적용 결과

정책 P 는 회사·도면 무관하게 작동하는 보편 신호만 쓴다. layer 이름
화이트리스트·키워드 매칭 금지, 외부 도구(DBSCAN·OCR·비전·Shapely) 도입 금지.

### 신호 1 — 일람표 영역 자동 검출

**알고리즘** (`poc_v2/tests/detect_table_region.py`, counter.py 미변경):

1. 자유 텍스트(TEXT/MTEXT) 좌표를 그리드 셀(셀 크기 = 도면 폭 × `region_size_ratio`)로 분할.
2. 점이 있는 셀을 8방향 인접으로 묶어 연결 영역으로 만든다 (셀 경계 걸침 보정).
3. 영역 판정: `서로 다른 부호 수 ≥ min_distinct_symbols` **AND**
   `부호당 최다 등장 ≤ max_count_per_symbol` → 일람표 후보.

**근거 — 왜 보편적인가:**
- 일람표는 "표"라는 정의상 종류별로 1~2번만 등장한다. 부재 실배치 영역은
  같은 부호가 수십 번 반복 → `max_count` 조건에서 자동 탈락한다.
- 검출 입력을 자유 텍스트(TEXT/MTEXT)로 한정한다. 부재 실배치는 블록
  인스턴스(INSERT/ATTRIB), 일람표는 자유 주석 텍스트 — 이 구분은 DXF
  **구조상의 차이**라 회사 이름과 무관하다. 블록 좌표까지 섞으면 부재
  밀집 영역이 일람표 셀을 오염시켜 검출이 원천적으로 불가능해진다.

파라미터(`config/symbol_rules.yaml`):
```yaml
table_region_detection:
  region_size_ratio: 0.033
  min_distinct_symbols: 4
  max_count_per_symbol: 2
```

### 신호 2 — 규격 안내 텍스트 식별

`match_symbol(text, whitelist, exclude_with_spec=...)` 에 옵션 추가:
- `'부호 + 공백/하이픈/줄바꿈(\P) + 규격'` 형태(예: `SC1\PH 350x175x7/11`,
  MTEXT 정리 후 `SC1 350x175x7/11`) → 카운트 제외.
- **기본값 `exclude_with_spec=False` 는 기존 동작 그대로** → 도면1·2 회귀 안전.
- 도면2 는 `exclude_with_spec` 를 끈다 — BR2 부분 일치(`BR2 L-80X80X7` → BR2)가
  깨지기 때문. 도면4 는 BR2 가 없어 안전하게 활성화.

### 신호 3 — 좌표 중복 진단 (자동 제거 X)

`diagnose_duplicate_coords(coords_by_symbol, tolerance_mm=1.0)` —
동일 좌표(1mm 이내)에 같은 부호가 2개 이상 겹친 곳을 찾아 **경고 메모만**
출력한다. 카운트는 그대로 두고 적산 전문가 검수 단계에 위임한다.

## C. 좌표 중복 진단 결과 (모든 도면)

| 도면 | 좌표 중복 현황 |
|---|---|
| 도면1 | 없음 (tolerance 1mm) |
| 도면2 | 없음 |
| 도면4 | **SB1 12곳 24개 (중복 12개)** |

**도면4 SB1 의 +12 차이를 좌표 중복이 정확히 설명한다:**
- SB1 ATTRIB 48개, 그 중 **서로 다른 좌표는 36개** (= 정답).
- 12개 좌표에 SB1 ATTRIB 가 정확히 2개씩 겹쳐 있다 → 중복 12개.
- 48(원시) − 12(중복) = 36(정답). 일람표 TEXT 1개까지 빼면 예측 48 vs 정답 36.

즉 SB1 의 오차는 매칭 룰 오류가 아니라 **DXF 자체의 엔티티 중복**이다.
자동 제거(중복 좌표 dedupe)는 진짜 인접 부재 2개일 가능성을 배제할 수 없어
1단계에서는 보류하고, 적산 전문가 검수 단계로 위임한다.

## E. 1단계 핵심 학습

- **"도면별 특성은 yaml 로"** — `text_height_filter`, `policy_p`,
  `table_region_detection` 모두 yaml. counter.py 에 도면명 하드코딩 0건.
  도면4 정책을 켜고 끄는 데 코드 수정이 필요 없었다.
- **"AI 는 결정만, 도구는 측정만"** — 일람표 검출은 ezdxf 좌표 분포라는
  결정론적 측정. LLM 호출 없음. 같은 입력 → 같은 출력.
- **catastrophic regression 방지** — 모든 변경을 `exclude_with_spec=False`
  기본값 뒤에 두어 도면1·2 회귀를 원천 차단. 회귀 테스트 29건이
  도면1 22건 무손상을 매 변경마다 보증했다.
- **보편 룰의 한계 인정** — layer 이름 매칭을 끊었더니 일람표/규격을
  "표의 구조"와 "텍스트 형태"로만 식별해야 했고, 그 결과가 오히려 회사
  무관하게 재사용 가능한 신호가 됐다.

### 멘토 미팅 의제
1. 1단계 종료 선언 승인
2. 2단계(부호 → 규격 매칭) 킥오프 — 도면4 `G-MEMBGUIDE` MTEXT 는
   1단계 제외 대상이지만 2단계엔 핵심 규격 데이터
3. 도면2 SC, 도면4 SB1 — 2단계 통합 도구 검토 후 재방문
