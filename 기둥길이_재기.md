Shapely : CAD 도면에서 "이 LINE 엔티티의 길이를 재라"는 작업에 잘 맞다. ezdxf만으로도 두 점 거리는 잴 수 있지만, Shapely는 폴리라인·교차·근접도 같은 더 복잡한 기하 질의를 깔끔하게 해준다.

다만 멘토님이 "라인 인식 관련 이야기"에서 강조한 것은 라이브러리가 아니라 **매핑 문제**다. 핵심은 "부호 텍스트 → 그 부호가 가리키는 LINE/POLYLINE 엔티티"의 연결이다. 그 매핑만 풀리면 길이 자체는 Shapely든 ezdxf든 한 줄로 계산됨. 어려운 건 매핑이지 길이 측정 자체가 아님.

- **기둥 길이 인식 아이디어**
    
    ### **1차 자동 측정 → 사용자가 수정한 결과 → 룰 보강**
    
    자동 측정이 틀린 케이스에서 사용자가 어떻게 수정했는지 패턴을 라우팅 시트에 메모로 누적. 예를 들어 "이 도면은 가장 긴 수직선이 외곽선이라 두 번째 긴 걸 채택해야 함" 같은 사용자 메모가 쌓이면 그 패턴이 룰 카탈로그로 승격. 
    
    1단계의 "도면별 yaml override → 자동화 자리 정리" 패턴과 동일.
    
    ### **"도면 종류 자동 추론" 사이드 효과**
    
    이 알고리즘을 모든 도면에 돌려보면:
    
    - 수직선 0개 + 수평선 많음 → 평면도
    - 수직선 등길이 9개 → 골구도/입면도
    - 수직선 다양 + 부호 있음 → 단면도
    
    라는 부산물이 나온다. 시트명 키워드 매칭과 교차 검증해서 도면 종류 분류 정확도 ↑
    
    ### **"신뢰도 자동 분류"로 사람 부담 차등**
    
    검증 결과를 신뢰도별로 분류:
    
    - 군집 1개 + 9개 이상 같은 길이 + 의미 있는 레이어 → 자동 확정 (사람 확인 안 받음)
    - 군집 2개 이상이지만 가장 긴 게 명확 → 자동 추천, 사람 확인
    - 군집이 흩어지거나 부호와 매핑 모호 → 사람 직접 선택

**전제**: 작은 도면 1장의 영역(박스 좌표)이 이미 확정되어 있다고 가정. 멘토님이 "큰 도면별 작은 도면으로 나누기는 가능"이라고 하신 그 결과물.

**흐름**: 그 영역 안에서 코드가 자동으로

1. 모든 수직선 추출 → 길이 분포
2. 영역 안의 부호 텍스트와 매핑 시도
3. 부호가 없으면 가장 긴 수직선을 공통 기둥 길이로 채택
4. 어디서 그 길이를 뽑았는지 시각화 (도면 위에 빨간 선 표시 등)

**목적**: 사람이 CAD에서 일일이 마우스로 클릭해서 측정하는 번거로움 제거. 코드가 1차 후보를 제시하면 사람은 확인만.

- 도면별 특징을 파악하고 길이를 어떻게 산출할지 의논
- 작은 도면으로 잘라서 기둥 길이를 파악한다 해도 외곽선을 가장 긴 길이라고 착각해버릴 수 있다.
    - layer 이름으로 구분이 가능할지 확인 (`0`, `DEFPOINTS`, `TITLE`, `BORDER`, `FRAME`)
    - 부호 좌표를 기준으로 반경 N mm 안에 있는 수직선만 후보로 좁히면, 외곽선처럼 부호와 떨어진 선은 자연스럽게 빠진다.
    - 기둥은 보통 비슷한 길이로 여러 개 있어(같은 부호 = 같은 길이). 외곽선은 보통 다른 모든 선보다 압도적으로 김. 길이 히스토그램에서 압도적으로 튀는 1~2개만 외곽선 후보로 빼는 휴리스틱도 가능.
- 부호별 기둥 길이가 다르거나 선 옆에 부호가 가까운 거리에 붙어있다면, 당연히 부호별로 길이를 재서 매핑을 하는 게 맞다고 생각함
- 측정에 사용된 수직선만 빨간색 강조 괜찮은 것 같음. 대신 표시될 선 굵기를 좀 더 굵게 하고 추가로 양 끝점에 작은 십자(+) 마커 사용. 사람이 직접 검수해야 하니까. 부호는 동그라미로 표시하는 걸로 하기.
