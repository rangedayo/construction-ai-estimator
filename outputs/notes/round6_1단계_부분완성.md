## B. 자동 활성화 — 신호 2·3

라운드 5 까지 `policy_p` 에 도면별로 사람이 박던 정책 활성화 결정을,
`auto_policy.auto_detect_policy` 가 도면 분석만으로 자동 산출한다.
LLM·외부 라이브러리 없음(ezdxf 만). 같은 입력 → 같은 출력.

### 신호 2 — 일람표 영역 존재 → `exclude_table_regions`

`detect_table_regions` 재사용. **min_height 필터를 적용한 자유 텍스트**로
검출 → 검출 영역 ≥ 1 이면 True.

> min_height 적용이 핵심. height 필터 없이 검출하면 도면2 의 작은 글자
> 일람표가 1곳 잡혀 오판(True)한다. 필터 적용 후엔 0곳 → 정답 False.

### 신호 3 — 부호+규격 패턴 빈도 → `exclude_with_spec`

modelspace 텍스트를 `counter.match_symbol` 로 분류:
- 단독형 `"SC1"` ↔ 규격형 `"SC1 350x175x7/11"`
- 규격형 텍스트 수 ≥ 화이트리스트 부호 수 × 0.3 → True

**규격형 전용 부호 보호**: 어떤 부호가 '단독형 0개 + 규격형 1개 이상'이면
`exclude_with_spec=True` 시 그 부호 카운트가 0 이 된다 → 보호 발동(False 강제).

### 자동/수동 일치 검증 결과 (`verify_auto_policy.py`)

| 도면 | 일람표 | 규격형 텍스트 | 보호 | 자동 결정 | 라운드5 수동 | 일치 |
|---|---|---|---|---|---|---|
| 도면1 | 0곳 | 4개 (임계 6.6) | True `[BR2]` | table=F, spec=F | F, F | ✅ |
| 도면2 | 0곳 | 0개 (임계 1.8) | False | table=F, spec=F | F, F | ✅ |
| 도면4 | 1곳 | 18개 (임계 1.2) | False | table=T, spec=T | T, T | ✅ |

**도면1·2·4 자동 = 수동 완전 일치.** 임계값 임의 조정 없이 통과.

### BR2 보호 발동 도면 정정

라운드 6 이전 프롬프트는 "도면2 BR2 보호"라 했으나 **오류**. 정답지 확인 결과
BR2 는 도면1 정답지에만 있고 도면2엔 BR2 열이 없다. 보호가 발동하는 도면은
**도면1** — BR2 가 `BR2 L-80X80X7` 규격형으로만 등장(단독형 0개)하므로
`exclude_with_spec` 가 자동으로 False 강제된다. 결과는 동일하게 도면1 False.

### 새 도면 무손 적용 검증 (`simulate_new_drawing.py`)

도면1.dxf 를 `도면1_clone.dxf` 로 복사(정답지·`policy_override` 미등록).
`min_height` 한 줄(177)만 공급 → 자동 판단으로 카운트:

```
policy_override['도면1_clone'] = None  (자동 판단)
자동 판단: 일람표제외=False, 규격제외=False
카운트 결과: 22/22 통과, 총합 331 vs 331
```

→ **신규 도면은 min_height 한 줄만 추가하면 일람표·규격 정책은 자동 적용**된다.

## C. 미달성 — 신호 1(min_height) 수동 유지

라운드 6 사전 검증에서 **height 임계값의 answer-key-free 자동화가 구조적으로
불가능**함을 확인했다.

- 도면1 정답(22/22, 총 331)을 맞추려면 `min_height` 가 176~198 사이여야 하나,
  height 분포의 가장 큰 갭은 198→282(range의 46%)다. "가장 큰 갭" 알고리즘은
  `min_height ≈ 199` 를 산출 → 도면1 **20/22** → 절대 제약 위반.
- 실제 컷(176→198)의 갭은 range의 11%로 가장 큰 갭이 아니다. **어느 height
  무리가 진짜 부재인지는 갭 구조에 없고 정답지에만 있다.**
- 도면별 비율도 정반대: 도면2 는 mode의 0.75배 height 를 버리고, 도면4 는
  0.33배 height 를 남겨야 한다 → 단일 비율 룰 불가.

따라서 `text_height_filter` 는 yaml 수동 유지(라운드 5 값 그대로).

### 사용자 의문에 대한 답

- 새 도면 추가 시 **`min_height` yaml 한 줄**은 도면당 수동 추가 필요 = 사실.
- 신호 2·3(일람표·규격 정책)은 **자동** — 정책 활성화의 약 2/3가 자동화됨.
- height 완전 자동화는 외부 신호(정답지 합산·LLM·비전)가 필요.

## F. 1단계 핵심 학습

- **"AI 는 결정만, 도구는 측정만"** — 신호 2·3 은 결정론적 측정(ezdxf 좌표·
  텍스트 분류)으로 완전 자동화. LLM 호출 0건.
- **결정론 룰의 한계 발견** — 외부 신호 없이 안 풀리는 결정(신호 1)이 존재함을
  *코드 변경 전 사전 검증*으로 확인. LLM 도입의 정당성 근거가 됐다.
- **솔직한 한계 인정의 가치** — "가장 큰 갭" 알고리즘이 도면1 을 깨뜨림을
  구현 전에 발견하고, 임의 임계값을 박는 대신 멘토에게 방향을 물었다. 그
  결과가 "신호 1 수동 유지" 결정.
- **catastrophic regression 방지** — 모든 변경을 회귀 테스트 뒤에 두어 도면1
  22/22 무손상을 매 변경마다 보증.
