02 / 전략
도면마다 다른데, 어떻게 보편화했나

모든 도면에 그대로 통하는 규칙은 코드에 한 번만 박고, 도면마다 달라지는 값만 따로 떼어 설정(yaml)으로 분리했습니다. 이 경계선이 자동화의 핵심입니다.

보편 룰 — 코드에 고정

정답지 없이도 모든 도면에 그대로 작동. 회사가 달라도 안 바뀜.

  • 정규식부호·규격 패턴 (H형강 4세그먼트 등) — KS·표 형식은 회사 무관
  • 제외 prefixF·HD·THK·X·Y… 적산 외 부재 자동 차감
  • DIMENSION세로/가로 방향 판별 기하 — 도면 무관 보편

도면별 분기 — 설정으로 분리

도면마다 다른 값. 코드는 안 건드리고 yaml만 갈아끼움.

  • 길이 라우팅어느 단면도에서 어느 부호를 재는가 (도면마다 다름)
  • height 컷작은 글자=노이즈 거르는 높이 (도면1:177, 도면2:302…)
  • override자동 판단이 틀린 예외만 강제 — 현재 도면3에 "일람표 제외 ON" 한 줄

03 / 보완
자동화가 안 되는 부분은 어떻게 처리했나

"코드가 측정할 수 있는 결정"은 자동화로 흡수했고, "도면에 숫자로 안 적힌 지식"은 사람이 설정에 적되 코드가 그대로 재현하게 했습니다. 그리고 무엇을 해도 기존 통과 케이스는 깨지 않는 안전망을 깔았습니다.

자동화로 흡수

일람표 유무 판정

원래 도면마다 손으로 켜던 정책을 auto_policy.py가 도면 분석으로 자동 결정. 5개 중 4개는 자동이 정확해 설정 없음(null).

예외만 강제

도면3 override 한 줄

도면3 본체는 기둥·콘크리트가 C1/P1처럼 슬래시로 붙은 표기라, 자동 검출기가 본체 자유 텍스트를 일람표로 착각해 일람표를 0곳으로 오판 → 일람표 안 부호까지 셈. 그래서 exclude_table_regions: true(일람표 영역 제외)만 수동 강제.

안전망

회귀 테스트 절대 조건

신규 작업이 기존 PASS를 깨면 즉시 중단. 본선 코드(counter.py 등)는 무수정 보호.

설계 원칙 "AI는 결정만, 도구는 측정만." 코드는 숫자를 재고, 어느 값을 쓸지는 사람·LLM이 정한다.

회귀 테스트 — 새 도면/신규 작업이 들어와도 기존 통과가 안 깨지는지 검증하는 항목

신규 작업 전후로 항상 돌려 숫자가 떨어지지 않는지 확인합니다. 하나라도 깨지면 즉시 중단.

단계검증 항목통과 기준
개수부호별 합계 vs 정답지14/16 유지 (오차 5% 이내, 소수는 ±1)
길이부호별 길이 vs 정답지16/16 유지 (≤1000mm ±50, >1000mm ±2%)
규격추출 규격이 정답 규격을 포함하는지(도면·동·부호) 키별 정답 포함
규격적산 외 부재 미혼입P·BR·SBR·MF가 결과에 0건
규격보 부호 전수 보존SB·SG 등 누락 없이 존재
규격출처 분리 보존도면4 SC1이 1층·지붕층 2건으로 갈림

+ 본선 코드(counter.py 등)는 무수정 보호. 길이는 "어느 단면도가 어느 기둥인가"라는 도면 간 관계 지식이 필요해 length_routing.yaml에 사람이 명시 — 코드가 기하로 못 푸는 영역.

04 / 비전
다음 단계 — 딸깍 한 번으로 총중량까지

지금 yaml은 사람이 도면을 보고 채웁니다(새 도면당 5~10분). 다음 라운드는 이 자리를 LLM 라우팅이 채우게 합니다. yaml을 없애는 게 아니라, 채우는 주체를 사람에서 LLM으로 바꾸는 것 — 그래서 결정론적 재현성과 자동화가 동시에 성립합니다.

지금

사람이 yaml 작성

적산 담당이 도면을 보고 화이트리스트·라우팅·height를 직접 입력. 새 도면마다 5~10분 개입.

→
목표

LLM이 yaml 초안 생성 → 전문가 검토

LLM이 시트 종류·중복·부호 카테고리를 판별해 설정 초안을 쓰고, 전문가는 확인만. 코드는 그대로 결정론 실행.

LLM에게 주는 입력 — DXF만 던지는 게 아닙니다
①
측정 결과

코드가 먼저 뽑아 정리한 부호·규격·좌표. LLM이 DXF를 직접 파싱하지 않고, 완성된 추출기 출력을 받습니다.

②
yaml 스키마

채워야 할 빈칸의 형식. 자유 작문이 아니라 정해진 틀을 채우게 해 출력이 항상 코드가 읽는 형태입니다.

③
도면 규칙

부호 카테고리·시트 종류·표기 변형(H-/H /없음) 같은 도메인 규칙. 분량이 작아 프롬프트에 직접 주입.

※ RAG(외부 문서 임베딩 검색)는 현재 규모엔 과합니다 — 필요한 지식이 한두 페이지 규칙이라 프롬프트 주입이 더 정확·재현적. KS 단위중량표·회사별 표준사양서가 수백 항목으로 커지면 그때 RAG 검토.
